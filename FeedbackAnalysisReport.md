### [medium] · Data · 2026-05-10 (filed under "Other" in ClaudeFB.md)
**Feedback:** Parse error: Invalid Input Error: CSV Error on Line: 1583 · Original Line: China,CHN,2010,2425.47...

**File:** `src/DataStudio.jsx` (DuckDB CSV loader branch), `src/services/data/duckdb.js`

**Suggestion:** This is a DuckDB strict-type error: DuckDB auto-infers column types from the first N rows and rejects later rows that don't match (e.g., a numeric column that suddenly contains a string or an extra-long numeric literal). The fix is to pass `SAMPLE_SIZE=0` or `AUTO_DETECT=FALSE` with explicit `COLUMNS = {...}` when loading large CSVs via DuckDB, or use `all_varchar=1` to load everything as strings and let the pipeline's `type_cast` step handle conversion. Alternatively, wrap the DuckDB CSV `READ_CSV_AUTO` call in a try/catch and fall back to the JS `parseCSV` function (already at DataStudio.jsx:40) for files that fail DuckDB parsing.

**Invariants:** DuckDB is used as a performance optimization — the JS fallback path must remain fully functional. Never truncate `rawData` before passing to runner.js.
